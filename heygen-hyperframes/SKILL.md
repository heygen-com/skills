# HyperFrames — HTML Video Compositions

Write HTML. Render video. Built for agents.

A composition is an HTML file with `data-*` attributes for timing, GSAP timelines for animation, and CSS for styling. The framework handles clip visibility, media playback, and timeline sync.

**Requirements:** Node.js >= 22, FFmpeg. No HeyGen API key needed — rendering is local.

## When to Use HyperFrames vs Video Agent

| Need | Use |
|------|-----|
| Prompt-driven video ("make a video about X") | **heygen-video** (Video Agent API) |
| Frame-accurate control, deterministic output | **heygen-hyperframes** (this skill) |
| Animated data/charts, kinetic typography | **heygen-hyperframes** |
| Avatar presenter delivering a script | **heygen-video** |
| Multi-clip editing with precise transitions | **heygen-hyperframes** |
| Quick video from an idea | **heygen-video** |

**Combining both:** Use heygen-hyperframes for compositions that need precise control (title cards, data viz, transitions), then integrate avatar clips generated by heygen-video as media sources.

## Workflow

### 1. Scaffold

```bash
npx hyperframes init my-video                        # interactive wizard
npx hyperframes init my-video --example warm-grain   # from template
npx hyperframes init my-video --non-interactive      # CI/agent mode
```

Templates: `blank`, `warm-grain`, `play-mode`, `swiss-grid`, `vignelli`, `decision-tree`, `kinetic-type`, `product-promo`, `nyt-graph`.

With media: `--video clip.mp4`, `--audio track.mp3`.

### 2. Compose

Author HTML with `data-*` attributes for timing and layering, CSS for layout, GSAP for animation.

**Approach — think before coding:**
1. **What** — narrative arc, key moments, emotional beats
2. **Structure** — how many scenes, which tracks carry what
3. **Timing** — which clips drive duration, where transitions land
4. **Layout** — build the end-state first (CSS), then animate into it (GSAP)

**Composition authoring rules, data attributes, timeline contract** -> [../references/hyperframes-authoring.md](../references/hyperframes-authoring.md)

### 3. Lint & Validate

```bash
npx hyperframes lint              # structural validation
npx hyperframes validate          # + WCAG contrast audit
```

Always lint before preview. Catches missing `data-composition-id`, overlapping tracks, unregistered timelines.

### 4. Preview

```bash
npx hyperframes preview           # browser with live reload
```

### 5. Render

```bash
npx hyperframes render                          # standard MP4
npx hyperframes render --quality draft          # fast iteration
npx hyperframes render --fps 60 --quality high  # final delivery
npx hyperframes render --docker                 # byte-identical
```

**Full CLI reference** -> [../references/hyperframes-cli.md](../references/hyperframes-cli.md)

## Composition Basics

A minimal composition:

```html
<body>
  <div data-composition-id="my-video" data-width="1920" data-height="1080">
    <video id="clip-1" data-start="0" data-duration="5" data-track-index="0"
           src="intro.mp4" muted playsinline></video>
    <audio id="music" data-start="0" data-duration="9" data-track-index="2"
           src="music.wav" data-volume="0.5"></audio>
    <div id="title" data-start="1" data-duration="3" data-track-index="1">
      <h1>Hello World</h1>
    </div>
    <style>/* CSS here */</style>
    <script src="https://cdn.jsdelivr.net/npm/gsap@3.14.2/dist/gsap.min.js"></script>
    <script>
      window.__timelines = window.__timelines || {};
      const tl = gsap.timeline({ paused: true });
      tl.from("#title h1", { y: 60, opacity: 0, duration: 0.6, ease: "power3.out" }, 0.3);
      window.__timelines["my-video"] = tl;
    </script>
  </div>
</body>
```

### Key Data Attributes

| Attribute | Required | Purpose |
|-----------|----------|---------|
| `data-composition-id` | Yes | Unique composition identifier |
| `data-start` | Yes | Start time in seconds |
| `data-duration` | Yes (img/div) | Duration in seconds |
| `data-track-index` | Yes | Layer ordering (same-track clips cannot overlap) |
| `data-width` / `data-height` | Yes (root) | Canvas dimensions |
| `data-volume` | No | Audio volume 0-1 |
| `data-composition-src` | No | Path to sub-composition HTML |

### Layout Before Animation

Position every element at its **most visible moment** first, as static CSS. Then animate INTO that position with `gsap.from()`. Never guess the final layout from animated start states.

### Timeline Contract

- All timelines start `{ paused: true }`
- Register every timeline: `window.__timelines["<composition-id>"] = tl`
- Duration comes from `data-duration`, not GSAP timeline length
- Video must be `muted playsinline` — audio is always a separate `<audio>` element

## Non-Negotiable Rules

1. **Deterministic** — no `Math.random()`, `Date.now()`, or time-based logic
2. **GSAP visual-only** — animate `opacity`, `x`, `y`, `scale`, `rotation`, `color`, transforms. Never `visibility`, `display`, or media playback
3. **No `repeat: -1`** — calculate exact repeat count from duration
4. **Synchronous timelines** — never build inside `async`/`await` or `setTimeout`
5. **Transitions required** — every multi-scene composition must use transitions between scenes. No jump cuts.
6. **Entrance-animate everything** — every element enters via `gsap.from()`. No element appears fully-formed.
7. **No exit animations before transitions** — the transition IS the exit. Only the final scene may fade to black.

## Animation

GSAP is the primary animation runtime. Key patterns:

- `gsap.from()` for entrances (animate INTO CSS position)
- `gsap.to()` for exits (final scene only)
- Position parameter for sequencing: `"<"` (with previous), `"+=0.2"` (after gap)
- Stagger for groups: `{ stagger: { amount: 0.3, from: "center" } }`

**Full GSAP reference** -> [../references/hyperframes-gsap.md](../references/hyperframes-gsap.md)

## Integration with HeyGen Skills

- **Avatar clips:** Generate avatar video with heygen-video, then use the MP4 as a `<video>` source in a HyperFrames composition for precise scene control
- **AVATAR files:** HyperFrames compositions can reference AVATAR-*.md files when the workflow involves combining avatar footage with custom compositions
- **Voice:** Use `npx hyperframes tts` for local TTS, or generate voice via heygen-video's voice pipeline

## Error Handling

- `npx hyperframes doctor` — check environment (Chrome, FFmpeg, Node, memory)
- Lint errors → fix before preview (missing data attributes, overlapping tracks)
- Render fails → run `doctor`, check FFmpeg installation
- Contrast warnings → adjust colors to meet WCAG 4.5:1 (normal) or 3:1 (large text)
